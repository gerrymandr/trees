8 """
9 deepcopy(x)
10
11 Create a deep copy of `x`: everything is copied recursively, resulting in a fully
12 independent object. For example, deep-copying an array produces a new array whose elements
13 are deep copies of the original elements. Calling `deepcopy` on an object should generally
14 have the same effect as serializing and then deserializing it.
15
16 While it isn't normally necessary, user-defined types can override the default `deepcopy`
17 behavior by defining a specialized version of the function
18 `deepcopy_internal(x::T, dict::IdDict)` (which shouldn't otherwise be used),
19 where `T` is the type to be specialized for, and `dict` keeps track of objects copied
20 so far within the recursion. Within the definition, `deepcopy_internal` should be used
21 in place of `deepcopy`, and the `dict` variable should be
22 updated as appropriate before returning.
23 """
